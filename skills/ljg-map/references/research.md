# 行业建图 + 节点识别 + 三大问题（研究 playbook）

这是 ljg-map 的内容命门。卡好看是壳；能不能 deep research 出一个行业的真实结构、把价值流摊成地图、找准瓶颈与价值捕获节点、提炼三个大问题，才是命。失手则退化成豆瓣式行业百科或一张装饰流程图。

> **地基（《千脑智能》参考系理论）**：地图让不可见的结构变可见。研究的目标不是堆事实，是**找到这个行业真正的坐标系**——价值沿什么方向流、各环节钉在哪、流量在哪收窄、利润在哪沉淀。

## 一、deep research：要查什么

联网研究（走 Research / web-access skill，真查不凭印象）。围绕五组问题，每组都要落到「环节」上：

1. **价值链结构**：从原始投入到终端用户，价值经过哪些环节？谁喂给谁？（画出流向）
2. **各环节的玩家**：每个环节当下谁占着（公司/角色）？集中还是分散？——记住玩家是占位者，节点是功能位。
3. **利润分布**：钱最后沉在哪一环？哪环毛利高且守得住？哪环是苦力活、利润薄？（价值捕获的证据）
4. **产能/流量卡点**：要扩大产出，最先卡住哪一环？谁扩产难、谁单点控制、谁设了牌照/专利/标准闸？（瓶颈的证据）
5. **未决张力**：这个行业当下最大的争论、变量、转折点是什么？什么一旦定了会改写格局？（三大问题的素材）

> 给了用户自己的判断 → 当作先验，研究去验证/补全/挑战它；没给 → 走全程。
> 经典书是经典行业可借 LLM 内部知识，但每条都默问：这是查到的，还是我编的？编的删。

## 二、研究扇出纪律（务必遵守）

并发研究 agent 时（feedback_research_agent_fanout_synchronous）：
- **同步返回**：每个 agent prompt 明令同步返回结果，**禁挂 Monitor 后台漂着**。
- **批量 + 错峰**：查限流 API 批量发、错开时序，别同时砸。
- **ID 抽样复核**：落盘前对关键事实（谁占哪环、谁毛利多少、谁卡着）抽样独立复核，别让单一来源的数字直接进卡。
- 五组问题可分给 5 个并行 explorer，各查一组，回来合并去重。

## 三、选地图拓扑（按行业结构定）

一个行业的「真坐标系」决定地图长什么样。三选一：

| 拓扑 | 何时用 | 主轴 | 节点 | 瓶颈/价值捕获典型位置 |
|------|--------|------|------|----------------------|
| **价值流（默认）** | 制造/供应链/硬件型（半导体、新能源、咖啡、医药） | 上游→下游（左→右） | 价值链各环节 | 瓶颈=稀缺产能环；价值捕获=议价权环 |
| **分层技术栈** | 软件/AI/云（基建→模型→中间件→应用） | 底层→上层（下→上） | 技术栈各层 | 瓶颈=某关键底层；价值捕获=某高毛利层 |
| **生态网络** | 平台/双边市场（电商、外卖、社交、支付） | 中心平台 + 多边角色环绕 | 平台 + 各边角色 | 瓶颈=多边连接卡口；价值捕获=收租的中心 |

判据：**价值主要是「流过去」的 → 价值流；是「叠上去」的 → 分层栈；是「连起来」的 → 生态网。** 拿不准默认价值流。

## 四、钉节点：三步

1. **提炼主轴**：这个行业价值排序/流动的主维度是什么？（上下游 / 层级 / 中心-边缘）轴两端要有名字。
2. **钉位置**：把研究出的关键环节（功能位）钉到轴上的位置。每个节点旁标当下占位的公司/角色。位置要说得出几何理由——谁在谁上游、谁离终端近、谁可被替代。
3. **标流向**：用箭头画价值/产品的流动方向。

测试：每个关键环节都能在轴上找到唯一位置吗？摆不进去 = 轴提错了，重提。

## 五、标两类节点（独立两遍，再看错位）

**先分别独立找，不要假设它们重合。**

- **第一遍找瓶颈**（🔴）：问「要扩大产出最先卡哪环」，用 SKILL.md 的四类信号（稀缺产能/单点控制/牌照专利标准闸/物理认知极限）逐环排查。标 🔴。
- **第二遍找价值捕获**（🟡）：问「钱最后沉在哪环」，用三类信号（议价权/高毛利且持续/赢家通吃）逐环排查。标 🟡。
- **再看落点**：
  - 🔴🟡 落同一节点 → **重合**（权力与利润同源，如 ASML/台积电）。caption 点：卡脖子的就是赚钱的。
  - 🔴🟡 落不同节点 → **错位**（价值在 A 创造被卡、钱在 B 捕获）。**在图上画虚线把 A、B 连起来**，caption 点破：「价值在这创造，钱在那捕获」。

> 错位是这张卡最锋利的发现。优先把它做足——它就是这个行业的权力地图。

## 六、提炼三个大问题

地图画完，提炼**正好三个**决定行业未来的大问题。

**判定标准**（每个都要过）：
- **未决**：答案还没定。已成定论的不是大问题。
- **高杠杆**：一旦定了会改写格局（流向变、瓶颈被绕过、价值池迁移）。
- **可争论**：聪明人会在它上面分两派，不是科普题。
- **指向未来**：问的是「会不会 / 谁将 / 何时」，不是「是什么」。

**怎么找**：它们往往落在地图的前沿——
- 瓶颈前沿：这个卡点会被新技术绕过吗？（如 AI 算力瓶颈会被新架构/光芯片绕过吗）
- 价值池前沿：利润会从当前节点迁走吗？迁到哪？
- 流向前沿：哪个新玩家/新模式在改写价值流方向？
- 边界前沿：这个行业会和哪个相邻行业融合/被吞并？

三个之间不重叠，合起来盖住这个行业最关键的不确定性。每个一句话，锋利、可争论、指向未来。

测试：把三个问题拿给一个圈内人看，他会不会说「对，这就是我们天天吵的三件事」？会 → 抓到；摆三条科普常识 = 没找到，重提。

## 六·B、提炼三个 base rate（地图的刻度）

地图给结构，base rate 给刻度——这个行业三个最关键指标的基线值，是没有额外信息时该持有的先验。继刚的贝叶斯 stance：先验先于个案。

**选哪三个，判据三连（都要过）**：
- **决策相关**：知道这个数会改变你怎么下注。
- **反直觉**：大众脑中的印象偏离它（被个案/幸存者偏差带歪）。一个 base rate 的价值 = 它和大众锚定印象之间的落差。
- **可查证**：deep research 查得到真数字（区间也行），不编。

**每个三字段**：指标名 + 基准数值（区间/百分比，大字）+ 一句「这意味着」（点破它戳掉的那个幻觉）。

**怎么找**：问「这个行业里，外行最容易被个案带偏的三个数是什么？」——成功率/存活率（创业、新药、餐馆）、份额分配（谁拿走多少，如咖啡豆农 ~1–3% 零售价）、成本/价格曲线（训练成本、推理成本年降幅）、集中度（前几名吃掉多少）。挑落差最大、最能解毒的三个。

**查证**：基线数字必须来自可追溯来源，区间表达诚实（如「~10%」「1–3%」「$1亿+」）；多来源对一下，别让单一耸动数字进卡。编的 base rate 比没有更糟——它假装是先验却是幻觉。

## 六·C、头像「你在这里」位置判断

研究时顺带判一句：**看这张图的人（继刚/用户）在这张地图上有没有一个明确的位置？**
- 有——继刚/用户本身是这行的玩家，占着某个功能位节点（如继刚在「知识付费/AI 应用」这类行业里）；或用户明确问「我在哪」。→ 头像作「你在这里」定位针嵌在那个节点。
- 无——陌生行业的纯鸟瞰，看图人不在图里。→ 不放头像。
判据照搬 ljg-library 的纪律换尺度：画面里本来有没有「你」。画法见 `references/visual.md`。

## 七、反翻译腔 + 禁切痕风（卡上 prose 必过）

thesis / caption / 三大问题都是要上卡的中文。每句默念出声，卡的地方整句重写，问一句「汪曾祺会这么写吗」。
- 名词化是命根子：「做出决定」→「决定」；「价值的捕获」→「谁把钱赚走」。
- 介词链砍：「通过 X 实现 Y」→ 动词直接带宾。
- 禁切痕风：别用「这一刀」「锋利」（当元描述）「钉死」「砸实」预告重话。内容够力读者自己看得出。
