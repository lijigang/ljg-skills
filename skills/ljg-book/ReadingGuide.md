# Reading Guide：建立 x → f → f(x)

正文不是章节摘要，也不是只摘一句人生启示。它要重建一本书的主逻辑：作者看见了什么问题，给出什么回答，这个回答怎样改变我们对问题的处理。

先在内部完成 x、f、f(x)，确认三者连通，再开始写。

## 1. 先看材料

只给书名时，先拿到足以支持整条逻辑链的材料：

- 目录、导言、样章、全文或可核实的书内片段。
- 书目信息不稳时，再查作者或出版社页面。
- 只有当事实错误会改变 x 或 f 时，才查作者访谈、可靠书评或外部基线。

给了 PDF、正文、样章或旧笔记时，先读原文。研究负责排除误读，不负责展示工作量。

按材料决定判断强度：

- *完整拆书*：有全文或足够书内材料。
- *初拆*：只有目录、导言、样章、访谈或可靠书评。
- *假设版*：资料很薄，只能提出有标记的暂定理解。

材料等级放在末尾「资料校准」。

## 2. 定义 x：作者在处理什么问题

x 必须是一个可以回答的问题，不是主题名。

依次问：

- 哪个困境让作者非写这本书不可？
- 旧理解在哪个地方失灵？
- 全书反复返回的是哪个疑问？
- 如果换一本同类书，这句话还成立吗？若成立，继续收窄。

一本书可能有多个问题。选择最能组织全书的主 x；副问题只在帮助主线时进入正文。

## 3. 提炼 f：作者怎样回答

f 是作者对 x 的核心处理方式。它可能是：

- 一个概念或重新命名。
- 一个关键区分。
- 一个框架或模型。
- 一套方法或操作顺序。
- 一条因果机制。
- 一个反复出现的意象、人物关系或叙事运动。

不要只报出术语。把 f 写成一句能运转的话：

> 面对 x，不应继续用 A 理解，而应通过 B 看见 C。

f 可以有 2 到 4 个必要部件，但必须说明这些部件如何共同回答 x。无法删减的清单不等于框架；能解释关系才算。

## 4. 计算 f(x)：回答之后怎么办

把 f 放回 x，观察它改变什么：

- *看见*：以前忽略哪个信号，现在会看见什么。
- *判断*：以前用什么标准，现在改用什么标准。
- *提问*：以前先问什么，现在先问什么。
- *行动*：以前如何处理，现在调整哪个顺序或动作。
- *克制*：以前会自动做什么，现在先停在哪里。

f(x) 不是额外添加的个人感悟。它必须从 f 推出来。删掉 f 后，f(x) 若仍像一句普遍正确的话，说明写得太泛。

文学作品不必硬造步骤。f(x) 可以是一种新的容纳、注意或判断：以后再遇到某类经验，读者不再像过去那样急着命名它。

## 5. 用书内材料接起逻辑链

正文通常需要 2 到 4 个书内锚点：场景、实验、人物片段、案例、区分或比喻。

每个锚点必须有工作：

- 证明 x 确实是书的问题。
- 展示 f 的一个部件怎样运转。
- 说明 f(x) 为什么不是凭空添加的建议。

锚点不是章节清单。它们应从不同位置共同指向同一个 f。

## 6. 做理解阻力扫描

完成 f 后，不要直接进入正文。先问：

> 如果读者只能看懂一个机制，哪个机制一旦成形，其余概念会自动归位？

再问：

> 这个机制靠文字最容易懂，还是靠形状、空间、动态、对照更容易懂？

按思想本身的形状选择表征：

- 凸凹、分布、边际变化、空间关系：ASCII 曲线或形状图。
- 前后步骤、因果传导、状态变化：流程图或时间轴。
- 循环、反馈、自我强化：回路图。
- 层级、部分与整体：结构图。
- 两种机制反复对照：小表格。
- 一两句话已经足够：纯文字，不画图。

图不是额外摘要，而是 f 的一种表达。当思想本身具有形状、空间关系或动态变化时，省略图会迫使读者在脑中重建它，此时必须画。图必须让关系比 prose 更快被看见，并使读者只看图也能说出作者改变了哪个判断。

ASCII 图放进：

```org
#+begin_example
...
#+end_example
```

宽度不超过 80 字符，只画一张。保持形状不变形。若只是把三行摘要重新排版，或没有降低理解成本，不画。

## 7. 写正文

正文通常 900 到 1600 个汉字，不含文件头与资料校准。短书可以更短；复杂书可以略长，但不要为了完整而扩成全书综述。

### x：作者在讨论什么问题

写清问题为何真实、旧反应为何不够。用一个书内锚点让 x 落地，不写宽泛领域史。

### f：作者怎样回答

这是正文重心。解释 f 的必要部件、部件之间的关系，以及书中材料怎样让它成立。把理解阻力扫描选出的图或表放在最需要它的位置；通常在核心机制第一次解释之后，而不是机械地放在节末。

### f(x)：怎样回应这个世界

把 f 放回 x。说明理解、判断、提问、行动或克制具体发生什么变化。不要另造应用清单，只写由 f 直接推出的后果。

### 资料校准

列 2 到 5 条真正支撑主线的材料：

- 书内材料：全文、样章、目录或导言。
- 必要校准：只有确实影响 x 或 f 时才列作者材料或外部来源。
- 材料等级：完整拆书、初拆或假设版。

每条只写「来源 + 它支持什么」。不要在这里补思想史或审稿判断。

## 8. 输出格式

保存到：

`~/Documents/notes/{时间戳}--拆书-{书名}__book.org`

时间戳：

- `date +%Y%m%dT%H%M%S`
- `date "+%Y-%m-%d %a %H:%M"`

文件头：

```org
#+TITLE: 拆书：《{书名}》
#+SUBTITLE: {作者} | {一句话 f}
#+DATE: [{YYYY-MM-DD Day HH:MM}]
#+FILETAGS: :book:{领域}:
#+IDENTIFIER: {YYYYMMDDTHHMMSS}
```

## 9. 文风

简洁、直白，像人说话。允许完整展开逻辑，不允许靠术语堆出深度。

少用「作者认为」「第 N 章指出」「在这个意义上」「提供了一种可能性」。直接写问题、回答以及回答如何运转。

不要出现「来源、承重、贡献类型、挪动类型、原创性」等审稿标签，除非用户明确要求。

禁用旧口头禅：骨架抽出来、精神内核就在你手里、走两步、delta、钉死、这一刀、锋利、砸实、带走的那一件。

## 10. 交稿前检查

依次补完：

> 这本书追问 ______。

> 它的回答是 ______。

> 因而，下次面对这个问题，我会 ______。

如果第二句拿掉后第三句仍成立，f(x) 太泛。如果第一句换一本书也成立，x 太宽。如果第二句只剩一个术语，f 还没展开。

最后遮住正文，只看图或表：读者能否复述作者改变了哪个判断？不能就重画或删除。若核心思想本身具有形状、空间或动态关系而全文没有图，回到第 6 步。
