# ljg-book: 拆书

输入一本书，输出三件事：作者在答什么问题，用什么取景框答，答出了什么。

## 方法

一道等式：**看世界 = f(x)**。x 是作者面对的问题，f 是他看这个问题用的取景框，f(x) 是透过这副框看到的画面，加上他落下的结论。

一本认真写的书，是作者对一个问题的一次回答。这个问题早有人答过，有个大家默认的答案；作者觉得那个答案不对，这本书才存在。所以书的价值不看它说了多少，看它把认知挪动了多少：原来大家看见 X，作者换一副框，看见 Y。

拆书的中心裁决：**这副框是作者自己造的，还是从别处捡来的现成框？**

- **捡**：用一副已经有名字的框（行为经济学、演化论、系统论……）去照一个新对象。捡来的框也能写出好书，但贡献在用法，不在框本身。要指明从哪里捡的。
- **造**：搭出一副之前不存在的框。
- **捡+加工**：框是捡的，作者加了一道自己的工序——一张表、一套练习、一个跨领域的接口。Kleon 捡了「一切皆重组」，加的是让新手每天能练的动作。框记在别人账上，工序记在作者账上。

判造还是判捡，用 Deutsch 的标准：**hard-to-vary**。抽掉框里的一个部件，整个解释还成立吗？还成立，说明那个部件是装饰，框是松的，多半是捡的或者干脆是套话；一抽就垮，说明每个部件都在受力，这是亲手搭的。塔勒布的凸性，抽掉延森不等式就垮，是造的。

默认判捡。多数书是捡，或者捡+加工；真造出新框的书很少。hard-to-vary 过了、书外验证（第五步）也过了，才升为造。「造」是个事实判断，别当荣誉发。

## 成品的样子

正文之前，先给三行：

```
- *x*：这本书在答的问题
- *f*：作者的取景框（用书里自己的词），末尾一词裁决：造 / 捡 / 捡+加工
- *f(x)*：照出的画面 + 落下的结论
```

这三行是全文的索引。槽位固定，内容必须全部来自这本书，一句放之四海皆准的话都不能有。x、f、f(x) 三个记号全文只在这里出现，正文说人话，不用记号。

正文 4 到 6 节。小节标题从这本书里长出来，说这一节实际做的事，不用固定标题——旧版「一、核心问题／二、共识基线……」的六连号不许回来。节数随书调整：造的书可以把 f 拆成两节慢慢写，薄的书可以把基线并进第一节。次序不变：x → f → f(x)。

## 五步

### 第一步：找出 x

把问题从话题里剥出来。「科技与未来」是话题，「科技演化有没有自己的方向」才是问题。问题找错，后面全错。

找问题时，一并找出它的默认答案：这个问题上大家原本信什么，那个答案为什么开始松动。作者就是冲着它来的。这个答案第二步还要用。

从哪找：导言里「我写这本书是为了……」那句话；找不到，就看作者几百页里反复绕回去的那个地方。书名常常是答案的种子——《必然》二字，说给怕科技的人听：挡不住，焦虑没用。

自查：把这一节拿去描述同领域另一本书，还成立吗？成立就是写得太泛，改到具体为止。

### 第二步：摆出基线

要量挪动，先有零点。列出作者写书之前，这个问题上已有的 2 到 4 种答案，每种一行：谁（流派或人物），他们看见了什么。标出哪一种是作者要反对的共识——教科书立场、外行常识，或者领域里不言自明的那套。

找不到成型的流派，退一步问：读者翻开书之前默认怎么想？那就是基线。

基线必须正对作者的不满。摆对了，作者看见会说「对，我就是冲这个来的」；摆一堆和他无关的学术流派，等于没摆。

薄的书，这一步可以并进第一步。但不能省——没有基线，后面的挪动就没有参照。

### 第三步：说清 f，裁决造捡

两件事：先说清框，再下裁决。

说框，用这本书自己的词。框不一定是名词，可以是一条公式（Shannon 的 H = -Σp log p），一个区分（自然态与文化态），一个动作（还原到 root cause），也可以是一组对照。必须写出这本书独有的术语、清单或区分——别的书没有、可以直接引用的东西。没有它们，这一节就能套到同领域任何一本书上——那是把领域拆了一遍，这本书原封没动。

然后裁决：造、捡，还是捡+加工，标准在「方法」一节。捡的，点名从哪捡；捡+加工的，框和工序分开记；判造的，先真跑一遍 hard-to-vary——写出抽的是哪个部件、垮没垮。

「造」「捡」「hard-to-vary」是这份 skill 的工作语言，不进交付的正文。正文里下裁决，用这本书自己的形象说：这框是从某派借来的，或是作者亲手搭的。两本不同的书，裁决那句话不该长一样。

裁决落定，挪动自然出来：相对基线，作者把认知从哪挪到了哪。挪动分三种，据实判，不论高下：

- **修正**：共识大体对，作者补了一块。
- **倒转**：共识答反了，作者翻了过来。
- **换轴**：作者不在原来的维度上作答，另起一个维度，旧问题整个作废。换轴的书通常也是造的书——他的框在旧坐标系里没有位置。

多数书是修正。别往换轴上靠，那是奉承，不是看清。

篇幅跟着裁决走：造的书，这一步是全文重心，hard-to-vary 的检验可以展开写；捡的书，这一步点明来源就够，篇幅让给基线和 f(x)。

### 第四步：写出 f(x)

两半：画面，落点。

画面是戴上这副框之后世界的样子。清单型的书，清单放在这里（KK 的十二个动词、Kleon 的好偷坏偷对照）；单概念的书，围着那个概念把画面写透。

落点是合上书要带走的那句话。检验它：单独抽出来，脱离上下文，还站得住吗？复述书名不算落点；「事物都是辩证的」这种谁都能说的话不算落点。落点要具体到有得驳——「作者认为人性是复杂的」，什么不复杂？这种软话删掉。

能引作者原话最好，引不到就用最贴的复述。译文要过朗读关：「X 是 Y 的拙劣替代品」这种从英文直译过来的骨架，宁可意译也不留。

### 第五步：画参考系图，把框推到书外

前四步都在书里，这一步走出去。先画图，再验证。

画图的依据：人认识任何东西，都是把它放在参考系的某个位置上（《千脑智能》）。各家对同一个问题吵几十年，吵的往往不是事实，是站位。把所有人放进同一张图，谁挨着谁、谁对着谁、作者从哪挪到哪，一眼看清。挪动在文字里是一句话，在图上是一段看得见的距离。

画法四步：

1. **定轴。** 回到第二步的基线：各家真正吵的是哪个维度？那就是轴，两端各一个极。选什么维度当轴，本身就是立场——作者从来不问的那个维度，就是他不言自明的地基，图注里要点出来。一个维度画一条数轴就够，不硬凑十字。
2. **落点。** 流派用 ●，作者用 ★，旧共识文字标出。每个点说得出几何理由：谁和谁近（答案相似），谁和谁对角（针锋相对），谁在原点附近（默认立场）。作者的位置由轴决定，别顺手放右上角。
3. **标出挪动。** 旧共识到作者那段距离：往哪个方向，挪了多远。捡的书，作者常常只是在共用的轴上换了个位置；真换轴的书，作者落在旧轴之外甚至图外——「他的框在旧坐标系里没有位置」，画出来就是这个样子。
4. **书外验证。** 把框推到作者没去过、读者原本也不信的地方，用它押一个可能会错的预测，再对照现实。这是 Deutsch 说的 reach：当初造框时没想着让它管的地方，它够不够得着。够得着、预测还准，是真造的硬证据；一推就垮，或者只在原地成立，那就是捡，照实写。

书外验证，两条规矩：

- 预测必须可证伪。中了写中，半中写半中，落空写落空。诚实的落空，比假装的命中有用。
- 禁止把框推回继刚自己的概念（降秩、凸性、四轴、下行封死上行敞开），再补一句「你早就在这么做」。那是照镜子，照见的还是自己。要推向书外的世界。

收尾，把读者该记住的东西交出来：就是这副框本身，用这本书独有的术语说。怎么收，每本书自己找话。这一步的四个动作本本相同，话得每本新说——上一本用过的说法，这一本给同一个动作另找一句。

图的硬性要求（org 输出）：

- 包在 `#+begin_example` / `#+end_example` 里，宽度不超过 80 字符。
- 轴有名字，两端有极。● 流派、★ 作者、旧共识文字标注，三种记号不混。
- 图下配图注：每个点凭什么在那个位置，一点一行；挪动是哪种（修正／倒转／换轴）；末行点出作者不言自明的那个维度，那是他的盲点。
- 没有轴名、没有位置关系、没有挪动的框图是装饰，不算图。画不出轴，说明第二、三步没做透，回去重做。

常见的图形有三种，按这本书真实的分歧选，不照抄：单轴数轴（一维分歧，各家钉在线上）；十字坐标（两维分歧，四象限落位）；作者另起一轴（真换轴，新轴与旧轴垂直，或者画在旧图之外）。

## 文风

站位：拆书的人站在作者头上看。不夸，不贬，不辩护，不道歉。

语言：中文白话，汪曾祺、王小波、阿城那一路。短句多。「通过／借助／凭借／在 X 上」这类介词能省就省，动词直接带宾语。「作者论证了」「作者认为」「第 N 章指出」这类学术腔删掉，直接说作者做了什么。

段落的形状跟着内容变。「开头一句立论，展开两三句，破折号收个包袱」这个形状连用五段，读起来像节拍器。该短的地方一句独占一行，该展开的地方写足。破折号一段最多一个。

不给自己的话预告力度。「这一刀」「够锋利」「钉死」「砸实」这类修辞一个不留——内容有没有力，读者看得出来，不用作者吆喝。

每段写完默念一遍。卡住的地方整句重写，不是换个词。默念时问：一个没读过翻译小说的中国人会这么说吗？汪曾祺会这么写吗？

### 反翻译腔

| 翻译腔 | 中文 |
|--------|------|
| 让我们一起…… | 我们……（省「让」） |
| 提供支持 | 撑 |
| 做出选择／做出决定 | 选；定 |
| 在这个意义上 | 这么说 |
| 拥有……的能力 | 能…… |
| 进行（研究／讨论／对话） | 用具体动词 |
| 这片领域 | 这门学问 |
| 这是必要的 | 必须这样 |
| 一是……二是……三是…… | 散开写，不编号 |

句式上：少被动（「被这事困扰」改「这事缠着」）；少「是……的」（「这是重要的」改「这事要紧」）；动作别名词化（「做出判断」改「判断」，看见「X 过程」「X 行为」「X 方式」就警觉）；长定语拆句，两个「的」叠着就停下来重组；修饰语别堆，「深刻的、有意义的、重要的」留一个就够，「完全／极其／非常／彻底」这类装饰副词删；括号旁注删，要紧的写进句子，不要紧的扔；能省的连词、主语、「的」都省——中文靠意思自己粘，不靠语法标记。

表格和规则 grep 得到的是表面词，抓不到骨子里的翻译腔。那一层只有朗读能抓：默念，卡了就重写。

### 不许跨书复用的句架

下面这些句架，是这份 skill 从前拆书时用惯的，不止一本书里逐字重复过。交付的正文里出现任何一条，整段重写。禁的是句架照搬，不是禁背后的动作——检验、对照、收尾每本书都要做，但每本书用自己的话：

- 「前五段把骨架抽出来摆着，冷的」；「……的精神内核就在你手里」；「图能预测书外的事，它就成了你的」
- 「他冲的默认答案」；「撑不住了」当失效诊断的口头禅；「才动笔」「动笔前」
- 「delta 一句话」；「档位：」；「翻成人话」；「意思是：」；「为什么是这一件」；「一句话」当节奏填料
- 「走两步」当小标题；「骨架／抽骨架」当主导隐喻；「把 X 推到 Y，看它照多远」当节标题
- 「对个账」「对账，半中」当固定收束；「带走的那一件／该带走的就这一件」；「合上书……十年……只记一件」当固定收尾
- 「从仓库里捡／拿」；「抽掉……承重件……就塌／散架」；「这副框剥到底就一句／一个字装得下」当开场

## 材料

只给了书名：先 WebSearch / WebFetch 查作者、导言、目录和核心论点的解读；再查这个问题的认知史——作者之前各家怎么答，这是第二步的材料，搜「问题 + history of / schools of / debate / criticism」。拿到核心论证和基线再动笔。凭印象拆出来的是维基百科式空话；基线更编不得，编出来的流派过不了自查。

给了 PDF / arxiv / 正文：直接读，不用查。

经典书：可以凭记忆写，但每写一条问自己：书里真有，还是我编的？编的删。

## 落盘

1. 取时间戳：`date +%Y%m%dT%H%M%S` 和 `date "+%Y-%m-%d %a %H:%M"`。
2. 写入 `~/Documents/notes/{时间戳}--拆书-{书名}__book.org`。
3. org 格式：标题用 `*`，加粗用单星号 `*bold*`，不用 markdown 的 `**`。
4. 文件头（不变）：

```
#+TITLE: 拆书：《{书名}》
#+SUBTITLE: {作者} | {一句话核心观点}
#+DATE: [{YYYY-MM-DD Day HH:MM}]
#+FILETAGS: :book:{书的领域 e.g. philosophy/biology/economics}:
#+IDENTIFIER: {YYYYMMDDTHHMMSS}
```

5. 文件头之后、第一个 `*` 标题之前，放开头三行（x / f 带裁决 / f(x)）。
6. 正文 4 到 6 节，次序 x → f → f(x)，参考系图包在 example 块里。
7. 把文件路径报给用户。

## 交稿前自查

十二条，全过才交。自查在心里做，正文里不留痕迹；哪条不过，回去重写对应的节。

1. 开头三行齐全，f 行带裁决，三行内容全部来自这本书。
2. x 具体：这一节套不到同领域另一本书上；x 对着的默认答案点名了。
3. 基线摆了 2 到 4 家，标了共识，正对作者的不满。
4. f 用了这本书独有的术语／清单／区分，至少一个。
5. 裁决下了。默认捡；判造的，hard-to-vary 真跑过，写出抽了哪个部件、垮没垮。
6. 「造」「捡」「hard-to-vary」这些工作语言没进正文，裁决用这本书自己的话说。
7. 挪动据实判了修正／倒转／换轴，没往换轴上靠。
8. 落点单独立得住：能引，能反驳；不是书名复述，不是空套话，不是软话。
9. 参考系图真画了：有轴名、有极、有几何理由、有看得见的挪动；图注点了作者不言自明的那个维度。
10. 书外验证做了：推到书外具体的新位置，预测可证伪，对照结果照实报；没推回继刚自己的概念。
11. 文风过关：朗读不卡，反翻译腔扫过，没有预告力度的修辞，跨书句架一条没犯。
12. 全文不夸不贬不辩护；小节标题从书里长出来，不是固定模板。
