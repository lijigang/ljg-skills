# ljg-paper：把一篇论文讲成一个故事

一篇论文，看懂只算读了一半；讲给一个不懂这行的聪明人、讲到他能复述出来，才算读完。这个 skill 产出的就是这样一篇讲述：从头到尾一个连续的故事，主角是论文的命题——作者押上整篇论文去证的那句大胆猜想。

## 讲完要留下什么

听的人能复述五件事：

1. 困境——具体卡在哪（落在一个例子上），以及这个问题的行情：现有做法走到哪一步，一篇典型论文能往前挪多少
2. 命题——作者押的那句猜想，一句话说出来
3. 论证——机制怎么动，证据怎么撑，包括最反直觉的副发现
4. 视角——带得走的新看法：原来还能从这个角度看
5. 检验——命题放进生活哪里灵、哪里不灵；它要是对，接下来该看到什么

任何一件让听的人卡住，就是没讲好。正文不求短，求读者从听不懂走到能复述；凝练只在 title 一处讲究。

文章分两层：最上面一张速读卡，三句话给张地图；下面是完整的故事。卡片最后写、放在最前：读透全文，才压得出那三句。

## 故事的走法

七步，按顺序：

1. *主角* — 谁在这个故事里？领域里的研究者、某个模型、某个用户、一套系统，或者论文要回答的那个问题本身。开场两句把主角领上台。
2. *困境* — 主角面前卡着什么？解不了会怎样？读者要看到这事值得坐下来听完。
3. *旧路* — 前人怎么试的？他们并不笨——把卡住他们的两难摆出来，读者才信这堵墙是真的。这里顺带交代行情：现有做法走到哪、典型一篇论文能挪多少。后文所有数字都放回来比，读者才掂得出轻重。开创性问题没有前人，两难可以省，行情照交代——「以前没人敢碰」也算一种。
4. *转折* — 作者看到了别人没看到的什么？凝成命题：一句大胆到让人想反驳的话。整篇论文都在为这句话作证。
5. *解法* — 方法怎么把命题撑起来？机制、这么设计的理由，一步步在那个例子上铺开。讲完之后，读者能说出每个部件为命题出了什么力。
6. *结局* — 挑两三组最说明问题的数字，放回行情里比。最反直觉的副发现单独讲。
7. *内核* — 故事留下的那个新视角。说出这个角度，再说出它除了这篇论文还能看什么。

七步走完，纸内的故事收了；命题还要过三关才算完：博导审稿（论证站不站得住）、生活检验（哪里灵、哪里不灵）、往前推（它要是对，接下来该看到什么）。

这七步是节奏，不是标题。落到章节里：

| 故事的哪一步 | 写在哪一节 |
|------------|----------|
| 主角、困境、旧路（含行情） | 问题 |
| 转折的预告 | 问题的结尾 |
| 命题亮相、解法、结局 | 翻译 |
| 故事里的关键概念 | 核心概念 |
| 内核 | 洞见 |
| 三关：审稿、生活、预测 | 博导审稿、检验 |
| 接到读者自己身上 | 启发 |

速读卡不在表里。它是全文写完后压出来的。

### 一个例子

拿 LenVM（让模型一边写一边知道还要写多远）过一遍七步：

- 主角：一个写长答案的语言模型，正在写 GSM8K 的解题过程
- 困境：它不知道什么时候收尾——要么话没说完就停，要么啰嗦到 token 烧光
- 旧路：直接训一个长度上限，硬卡；一卡，推理质量就掉。行情也在这里：省 token 就得赔精度，没人两头占住
- 转折：命题——「还要写多远」本来就是一种剩余距离，可以像价值一样实时预测。模型每写一字都该知道还差多远
- 解法：RL 训练里加一个 length value head，用固定 -1 的 reward 让它学会「少写一字省一字」
- 结局：推理长度降 30%，准确率不降反升，两头占住了。副发现：token 词云里 wait/think 让位给 finalize/confirm
- 内核：「知道何时停」可以被监督出来——教模型收住，比教它答对更少有人做，也更值

命题走出论文的样子：生活里，台上有倒计时牌的人收得住，没有的人拖堂；往前推一两年，长度控制会从「卡上限」转向「预测剩余」。

成品里并没有这七个小标题。它们藏在「问题 → 翻译 → 核心概念 → 洞见」的连续叙述里，读者只觉得一个故事在走。

## 写作原则

1. *一条故事线走到底。* 每段末句自然引出下一段；章与章交界处，上一章收尾时读者已经想知道下一章的事。写完盖住所有章节标题从头读一遍，还像一个人在讲一件事才算过。
2. *一个例子撑全文。* 「问题」里立的那个具体例子，「翻译」「核心概念」都顺着它讲。换例子等于换地图，读者攒起来的直觉全丢。机制细节确实展不开时，可以加一个子例子，但必须在同一问题域里——读者感觉是拿放大镜看局部，不是换了个话题。
3. *带读者一起想。* 写「既然 A 是 B，那 C 能不能也是 D？」，让读者觉得结论差一步就是自己推出来的。别端出想完之后的结果。
4. *变形代替定义。* 讲两个概念的关系，把 A 一步步变形成 B 演给读者看。「把 LSTM 变形，看，像 ResNet 了」比「LSTM 和 ResNet 是对偶的」管用得多。
5. *结尾给能动手的事。* 「这意味着你可以___」，别停在「这让我们重新思考___」。

工具箱（都可选）：

- *类比* — 要扛得住细讲：方法的关键部件都能在类比里找到对应，讲解能沿着类比走完
- *ASCII 图* — 读者有了概念基础再画，展示部件关系、数据流、结构对比
- *前后对比* — 「以前这么想，现在该这么想」并排摆
- *好问题* — 把论文的难处变成外行也好奇的问题
- *递进例子* — 从简单到复杂，一步步搭理解
- *反问* — 碰到隐含假设，用问题打开

## 红线

1. *口语* — 你会这样跟朋友讲一篇论文吗？学术腔是默认的敌人。「本文提出了一种新的框架」要写成「他们做了个东西」。
2. *术语先落地* — 先用大白话说清，再顺带给术语名。必须用原文术语才解释得了，说明自己还没懂。
3. *一句一事* — 每句只推一步。多用短词和实词，形容词能省就省。
4. *不填充* — 「近年来随着……的发展」「值得注意的是」这类套话全删。
5. *开头给理由* — 问题节第一句要让人想知道答案。
6. *信任读者* — 说过一遍不再重复。唯一例外是速读卡：卡片给方向，正文给细节，两处不用同一句话。
7. *诚实* — 论文有硬伤就说有硬伤，看不懂的地方就说看不懂。
8. *半年后的自己看得懂* — 术语首次出现给解释（「value function = 给定当前状态预测未来累积奖励」），公式翻成自然语言，引用说明对外行的意义。写完默想：半年后在 denote 里搜到这篇，30 秒内能想起核心吗？不能就重写。
9. *外行优先于凝练* — 该铺垫就铺垫，篇幅由讲懂的需要决定。
10. *故事一气* — 整篇读下来像一个人在讲一件事，章与章之间由故事自己的牵引接住。

## 输出格式

章节名固定：速读 / 问题 / 翻译 / 核心概念 / 洞见 / 博导审稿 / 检验 / 启发。故事感靠写法和衔接传出来；标题不沾戏剧腔，不用「第一幕」这类叫法，protagonist、climax 这类英文戏剧词不出现在输出里。

Org 语法：加粗用 `*bold*` 单星号，禁止 `**bold**`；标题层级从 `*` 开始，不跳级。图表用纯 ASCII 字符（`+ - | / \ > < v ^ * = ~ . : # [ ] ( ) _ , ; ! ' "` 和空格），不用 Unicode 绘图符号。

结构以 `references/template.org` 为准。不要参考 `~/Documents/notes/` 里已有论文笔记的结构——旧文件可能是过期模板。

文件规范（denote）：

- 时间戳 `date +%Y%m%dT%H%M%S`；可读时间 `date "+%Y-%m-%d %a %H:%M"`
- 文件名 `{时间戳}--paper-{简短标题}__paper.org`，简短标题用方法名或核心概念名（如 `prerl`），供检索用；title 才是内容的入口，两者各干各的
- 写入 `~/Documents/notes/`

文件头：

```
#+title:      {一句话提出论文的核心思想或发现}
#+subtitle:   {论文原始标题，通常英文}
#+date:       [{YYYY-MM-DD Day HH:MM}]
#+filetags:   :paper:
#+identifier: {YYYYMMDDTHHMMSS}
#+source:     {URL 或来源描述}
#+authors:    {作者列表}
#+venue:      {发表场所/年份}
```

### title

title 是整篇的门面：读者扫一眼，就知道能从这篇论文带走什么。写完「洞见」再回头写它——从那段话里提出核心，炼成一句。

规则按优先级排：

1. *地道中文的凝练* — 像汪曾祺、王小波、阿城的标题：短、净。写完默问「一个没读过翻译小说的中国人会这样说吗」，不像就重写。反翻译腔自查表在 `~/.claude/PAI/USER/AI_WRITING_PATTERNS.md`（Layer A 必扫）。避开：被动句（「被锁在…里」）、「是…的」句、长定语（「那个由…引起的…」）、「进行+名词」、「让我们…」。
2. *不夹英文* — RL、Agent、token 都不进 title，术语留给正文。例外：人名、产品名（GPT、Claude）。
3. *6-15 字* — 超过 15 字就是还没炼到位，回去再炼。
4. *动词做骨* — 「重大的」「根本的」「惊人的」这类形容词全删，每个字都得干活。
5. *自带张力，三选一* — 反直觉（学会反成枷锁）；对仗并置（记得太牢，想不出新）；转折反讽（答得越准，看见越少）。
6. *不复述论文题目，不当方法名用* — 「关于强化学习预训练空间转移的探索性研究」和「PreRL：把强化学习搬进预训练空间」都不行。

正例（看转化方向）：

| 论文核心思想 | ✗ 翻译腔 / 口语化 | ✓ 中文凝练 |
|------------|------------------|-----------|
| 奖励信号把模型锁在已会轨迹里，擦掉题目就解锁 | 奖励信号会把模型锁死在已会的轨迹里 | 学会，反成枷锁 |
| 只用错样本做 RL，反思能力自己长出来 | 只用错样本做 RL，反思能力自己长出来 | 错处长出反省 |
| 模型在向量空间里思考比生成 token 更省更准 | 模型在向量里想，比一边想一边写更省更准 | 默想胜出口 |
| 多智能体缺的是组织协调而非个体智能 | Multi-agent 缺的不是聪明，是 HR | 多智不如善织 |
| 老师与学生视角不同导致教学失败 | 老师比学生高分还教不会 | 高分难为师 |
| 预测剩余长度让模型知道何时收尾 | 把"还要写多长"做成一个值函数 | 知止方能落笔 |

正例的共性：主干多在 4-8 字，节奏接近成语或古文标题；动词结实（锁、长、默、织、止）；有古意，不晦涩，不抒情。

写完三问：贴出来像不像一句篇名？中文母语者会这样说吗？删掉任何一个字塌不塌？有一个 No 就重写。

再拿给外人认一认：把 title 单独给一个没读过这篇论文的人看，问「这论文大概讲什么」。答得出方向就过；完全猜不到（高度凝练的古文式短句常这样），必须用中文 subtitle 兜底解释——英文原标题不算兜底：

```
#+title:      字未出，止已现
#+subtitle:   把"还要写多远"做成一个 value 函数 — Length Value Model: ...
```

凝练和认得出，两关都得过：只凝练，半年后自己都认不出是哪篇；只顾认得出，就退回二十八字的长句。

## 步骤

### 1. 拿到论文

- arxiv URL → WebFetch；PDF → Read（注意 pages 参数限制）；本地文件 → Read；只有论文名 → WebSearch
- 要拿到：标题、作者、摘要、核心方法、结果

论文若有一张一眼看懂全文的总览图（多半是 Figure 1），存到 `~/Documents/notes/images/`，命名 `{identifier}--paper-{简短标题}-overview.png`。arxiv 走 HTML 版（`arxiv.org/html/...`）找图片 URL 下载；PDF 截含图页面。没有这样的图就跳过，不硬找。

### 2. 问题

让读者亲历困境。开场给一个具体例子：一段输入输出、一个场景、一次失败，读者在例子上撞见问题。例子要一两句话说得完，越简单越好。

不写「大模型在事实回答上存在幻觉问题」，写「你问 GPT『2023 年图灵奖得主是谁？』它一本正经回答 Yann LeCun。再问一次，它说 Hinton。第三次又变了。」

一段连续叙述写三件事，同一个例子贯穿：

1. 主角与困境：谁、卡在哪、解不了会怎样
2. 旧路与行情：前人在这个例子上怎么做的，卡住他们的两难是什么；现状到哪、典型一步多大，顺带交代清楚
3. 转折的预告：作者看到了什么入口。只引方向，机制不展开，命题整句留给「翻译」开头

这一节不加子标题——子标题会把流切成段。子标题留给「翻译」用。

反例：「本文提出了一种新的 XXX 框架」——这是摘要的写法，进不了故事。

### 3. 翻译

「问题」已经把读者带到转折跟前，这一节不回头重述问题。

开头先亮命题：作者押的那句猜想，写成让人想反驳的一句。整节就是看作者怎么把这句话撑起来。

接着让读者看见方法的样子：在刚才那个图灵奖问题上，这方法像个分诊台，先把拿不准的回答拦下来。往后一步步展开，每段推进一步，前一段的话头开出后一段。揭示的口吻：「这时候你看……」「于是发生了……」「那这一步怎么办呢——他们想到……」。

仍然用「问题」里那个例子。要覆盖四件事，都落在例子上：命题是什么；机制怎么动（讲完读者能说出每个部件为命题出了什么力）；结果长什么样（两三组数字放回行情比）；理解全文需要的关键概念。子标题按内容组织，不必固定。

五条硬要求：

1. *类比要扛得住细讲* — 方法的关键部件都映射得上，能沿着类比讲完机制。范本：「跑步选手 + 手表」，模型对应选手，value head 对应手表，整套机制讲得下来。浮在表面的装饰类比不如不用。
2. *至少三组数字，每组都带参照* — baseline、改进、关键 ablation，每个数字都能回答「这在领域里算大算小」。没有参照的数字，读者称不出轻重。
3. *反直觉的副发现* — 论文里最让人意外的一段，单独成节。有就必须保留；没有就明说「这篇没有」，不硬挤。
4. *论证提得出来* — 讲完之后，读者能用三句话说出命题靠什么撑着：机制一句、证据一句、消融一句。提不出来，这节还没讲完。
5. *不放公式* — LaTeX 一律不进主文。要么翻成话（「L_len = … 意思是模型每写一字都被监督一次」），要么放进可选的附录节。

### 4. 核心概念

挑三个不懂就跟不上故事的概念（方法名、部件、数学对象、新定义），逐个讲。

三个是下限。只挑得出两个，多半漏了藏在方法里的关键设计取舍。LenVM 的 constant -1 reward 就是这种：算不上部件，却是让整套机制转起来的设计，得单独讲。真的只有两个（很少见），明说，不硬凑。

每个概念回到「问题」那个例子上讲：

- 一句话说清是什么、干什么用
- 在例子上长什么样，少了它例子会怎样；讲关系优先用「把 A 变形成 B」
- 少了它，论文的推理断在哪

「翻译」里已经讲透的不重复挑。

### 5. 洞见

故事真正留下的往往就一点：一个新视角，原来还能从这个角度看。

一句话说出这个角度，再补一句它除了这篇论文还能看什么。读者要觉得「这个想法我带得走」。

检验：把这句话抽出来，脱离论文的上下文，它还有没有力量。复述论文结论不算洞见；洞见是听完故事自己看见的东西，论文未必直说，但逻辑指向它。

说不出来就重读「翻译」。论文确实没有思想上的新东西，就直说「这是工程改进，没有认知层面的新发现」。

### 6. 博导审稿

换身份：在这个方向带了二十年研究生的博导。学生拿这篇论文来问值不值得认真对待。用办公室聊天的口气：

- *选题* — 那个困境是真缺口，还是人造缺口？
- *方法* — 巧劲还是蛮力？有没有更自然的路被跳过？重点找方法的根本预设：作者假设了什么而没有讨论。LenVM 假设 value 信号去调整采样分布之后，分布还稳定——可调整一旦发生，分布就变了，value 预测可能失准。这类自我指涉的隐患，比「规模不够大」之类的批评致命得多。自问：这论文如果错，最可能错在哪一步？根源是不是一个没人提的预设？
- *实验* — baseline 公道吗？消融做够了吗？那几个数字经得起追问吗？
- *声称的分量* — 放回行情称。「解决了 X」这类话，这个领域一年出多少篇？历史上几篇真站住了？超常的声称要超常的证据，别陪作者激动。
- *写作* — 最该说清楚的地方有没有偷懒？
- *判决* — strong accept / weak accept / borderline / weak reject / strong reject，一句话理由。

好的说好，差的说差在哪。末尾一句把话头递给下一节：命题在纸内站住了（或者没站稳），拿到纸外试试。

### 7. 检验

博导管纸内。这一节把命题带到纸外，走两步。

第一步，放进生活。挑一个读者熟到不用解释的场景，让命题在里面走一遍。哪里灵：台上有倒计时牌的人收得住。哪里不灵：人看见倒计时会慌，模型不会，「慌」在命题之外。不灵的地方是命题的边界；摸到边界，读者才算真的拿到这个命题，而不只是背下它。只用一个主场景，至多加一个对照。命题离生活太远（纯数学构造、底层硬件），明说测不了，不硬测。

第二步，往前推。命题要是对，X 条件下该看到 Y。三个方向至少推一条：

- 领域：往后一两年该冒出什么样的工作
- 现实：论文外的系统或人，什么条件下该观察到什么
- 产品：什么工具会变样

预测要具体到能错：有条件，有可观察的结果。「未来会更重视 X」这种错不了的话删掉。真推不出预测也直说：推不出预测的命题多半还没立住，这本身就值得写下来。

这一节管信不信，「启发」管用不用。

### 8. 启发

把故事接到读者自己的生活里。三个方向试：

- *迁移* — 论文的机制或视角，能移植给我体系里的哪个零件？具体怎么接？
- *混搭* — 论文的部件和我已有的东西组合，能出什么新的？
- *反转* — 论文的做法和我的默认假设相反吗？该停什么、开始什么？

命中的展开，没命中的跳过，全落空就写「没有」。落点是「你可以___」。

### 9. 速读

全文写完，回头压一张卡放到最前面。三句一句比一句细，开头一句引语把它们拢住：下面的长故事，就是把这三句展开。

- *一句话* — 平实说完整篇：谁卡在什么困境、作者押了什么、结果怎样。跟 title 分工：title 是炼出来的短句（需要 subtitle 兜底），这句是谁读一遍都知道「这论文干了什么」的话。两处别用同一句。
- *大想法* — 只能带走一个想法的话，是哪个。和「洞见」是同一个核：这里先给方向，读者带着目的地往下走；「洞见」在故事走完后落地，并给出第二用途。两处措辞不同。
- *只记三件事* — 半年后忘掉大半，这三条还在：通常是困境为什么值得解、命题靠什么撑住、走出论文带走什么。这三条是重建整个故事的支点；预测在「检验」，动手清单在「启发」，都不放这里。

### 10. 自查

过一遍红线，另加：

- 否定式排比（「不是 X，是 Y」）全文至多两处；三段式排比改成两项或四项
- 长短句交替，别一个节奏到底
- 听起来像金句的，改平实
- 逻辑每步能追上，没有跳步
- 速读卡：三句是同一篇的三个深度吗？每句在正文里找得到根吗？「大想法」跟「洞见」撞词没有？「三件事」跟「检验」「启发」串味没有？
- 盖住所有章节标题从头读一遍：还像一个人在讲一件事吗？章与章交界，上一章末句有没有让读者想翻下一章？七步和三关都走全了吗？命题在「翻译」开头亮了吗？行情交代了、结果的数字放回去比了吗？

查出的问题列成清单，逐条改完再写文件。

### 11. 写文件

按 denote 规范取时间戳，读 `references/template.org`，写入 `~/Documents/notes/`，报告路径。

## 验收

- 通读像一个人在讲一件事，不像十段汇报拼起来
- 七步齐：主角 / 困境 / 旧路 / 转折 / 解法 / 结局 / 内核（开创性论文可免两难，行情照交代）
- 命题一句话可指认，单独抽出来让人想反驳
- 行情先交代，结果的数字放回去比；全文没有无参照的数字
- 同一个例子贯穿问题 / 翻译 / 核心概念
- 翻译节五条硬要求全过：类比扛得住细讲、三组带参照的数字、反直觉副发现、论证三句提得出、无公式
- 核心概念 ≥3 个，含至少一个设计取舍
- 洞见抽出来脱离论文仍有力量，给了第二用途
- 博导指出至少一个没被讨论的根本预设，声称称过行情，有判决
- 检验两步落地：生活场景有灵有不灵，预测具体到能错
- 启发落在「你可以___」，与「检验」不重复
- title 三问全过，外人认得出方向（认不出就上中文 subtitle 兜底）
- 速读三句齐，每句在正文找得到根，与 title、洞见、检验、启发都不撞
- 零英文戏剧术语，主文零 LaTeX 公式
