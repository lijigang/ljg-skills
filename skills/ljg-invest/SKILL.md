# 投资报告

生成一份投资分析报告。核心只问一个问题：这个东西在创造新秩序，还是在搬运旧秩序？

## 认知起点

财富不是钱，是被欲望照亮的秩序。投资就是拿手里的秩序去换一台更好的秩序生成器。

所以不称重，看相：
- 不问"这个公司值多少钱"，问"这台机器转不转得起来"
- 不问"市场多大"，问"市场在用什么过时的眼睛看它"
- 不问"能涨多少"，问"我拿什么换什么，换完之后谁更聪明"

## 输入

公司名称、BP、文字介绍、对话记录，或任何描述项目的材料。知名公司只给名字即可——用 Research skill 或 subagent 抓取最新财报和行业数据。

## 报告结构

先过区块〇的法眼闸门——决定这家公司值不值得拆。过了，再展开下面五个区块；没过，一段话出局。

以下五个区块是骨架，不是填空题。对某个项目来说哪个区块最有料，那个区块就多写；没料的一两句带过或直接跳过。报告为判断服务，不为完整性服务。

<!-- ╔═══ 自定义：法眼闸门（2026-06-25）═══ -->

### 〇、法眼闸门（去留判定）

这一关只做一个动作：**决定这家公司要不要往下拆**。不命名、不展开——命名是区块一的活。这里只发一张准入证，或一张逐客令。

不称重，看相。第一眼只问：**它有没有一个让人想往下拆的"相"？**

那个相——一种说不清、却拽着你往下看的异常、张力、或不对劲。它具体叫什么，留给区块一去拧成精确命名；这里只判断它在不在。

二选一，不许骑墙：

- **相在** → 一句话点出它在哪，发准入证，进区块一正式展开。
- **相不在** → 它只是把现成的东西重新排列，生不出新秩序。一句话说清它搬运的是什么，**当场停笔，整份报告就这一段。**

原本五个区块默认每家公司都拆到底、硬挤一个结论。这道门补的就是这个缺口：有些公司，正确的结论就是"不值得有结论"。

### 一、这是什么

一张表 + 一句自定义赛道定义。

| 维度     | 内容                                         |
|----------|----------------------------------------------|
| 项目名称 |                                              |
| 赛道定义 | 用我们自己的语言，不用市场标签               |
| 阶段     |                                              |
| 融资情况 | 金额 / 估值 / 条款（有则填，无则标注）       |
| 数据快照 | 关键运营数据                                 |

赛道定义要穿透表面标签。"搜索引擎公司"是隔的，"人类认知基础设施的垄断运营商"是不隔的——后者告诉你它真正在做什么。

### 二、秩序创造机器判定

这是整份报告的心脏。不逐项打分，而是回答一个问题：**这台机器转不转得起来？**

从三个角度看：

**飞轮在不在转？**
系统有没有越用越好的结构——用户越多数据越多，数据越多产品越好，产品越好用户越多？这个循环是已经在转、刚开始、还是停着？转了多久？加速还是匀速？如果不转，什么卡住了？

**冲击后变强还是变弱？**
竞争来了、技术变了、市场塌了——这台机器是碎掉、扛住、还是吃掉冲击变成自己的燃料？历史上有没有被冲击过？结果如何？

**资源是被推过来的，还是自己来的？**
扩张靠的是一个一个谈、一块一块买（推），还是别人主动涌过来、因为不来就亏（引力）？有没有"不推而聚"的迹象？

**综合判定**：
- 秩序创造机器 — 飞轮在转，冲击后变强，资源自己来
- 有潜力 — 飞轮有结构但还没验证转起来
- 秩序搬运 — 把已有的东西重新排列，没有生成新秩序

### 三、创生公式

每台秩序创造机器都有一个核心算法。用一句话写出来。

参考：
- 亚马逊 = 利润→再投资→降成本→降价→更多用户→更多利润
- 特斯拉 = 硬件采数据→数据训练算法→算法重新定义硬件
- Google = 每次人类找答案的方式迁移时，成为新方式的默认基础设施

然后回答：
- 这个公式被验证过几次？验证到什么程度？
- 别人在跑相似的公式吗？差异在哪？

### 四、市场看见的 vs 我们看见的

这一节决定投资时机。

**它在 S 曲线的哪里？**
积累期、拐点、加速期、平台期。如果在拐点前——什么条件能触发拐点？

**市场在用什么旧眼睛看它？**
市场给它贴的标签是什么？这个标签遮住了什么？我们的框架看到了什么市场看不到的？认知差有多大——这是超额收益的来源。

三个信号检测认知折价：
- 需要很复杂的解释别人才能听懂？
- 定价持续异常（部分之和 ≠ 整体）？
- 已有的类比全部失效（"像 X 但又不像 X"）？

**它控制了什么别人拿不走的东西？**
权力来源是什么——数据、分发、标准、网络效应？这种控制是静态的（品牌、专利）还是动态的（越变越强）？
这个稀缺性未来会不会位移？项目有没有能力跟着位移走？

**它在搭哪趟便车？**
三种成本正在坍缩——理解成本、协作成本、行动成本。这个项目骑在哪一种上？坍缩释放的能量被它捕获了多少？

### 四·五、巴菲特三闸复核（可交易标的适用）

区块二到四回答"这台机器好不好"；这一关回答"**现在这个价格、以我的认知，配不配下注**"。仅对有市场报价的标的（上市公司等）执行；一级市场项目跳过本节，直接进区块五。

原则层来自巴菲特股东信全集降秩（一锚三闸，概念卡见 Obsidian `投资分析/巴菲特体系/`），三闸逐一给读数，不许骑墙：

- **锚**：伊索三问答得出吗——丛里有鸟？何时出来几只？无风险利率多少？owner earnings 可估吗？答不出 → 本标的属投机，直接出局，后面不用看
- **竞争闸（确定性读数上半）**：十年后护城河更宽还是更窄，凭什么？快变行业（护城河需不断重建）→ 读数直接给低
- **认知闸（确定性读数下半 + 折扣要求）**：这个判断在用户能力圈内还是圈边？保守估算下价格比价值低多少——只是略低 → 不买
- **情绪闸（折扣读数）**：当前报价是市场先生亢奋价还是抑郁价？买入理由是"价值高于价格"还是"价格在涨"？

**四象限定格**（纵轴=确定性：竞争闸×认知闸；横轴=折扣：情绪闸）：

| 格子 | 判定 | 动作 |
|---|---|---|
| 黄金格 | 高确定 + 大折扣 | 可重注，退出只看护城河与高估，不看股价 |
| 等待格 | 高确定 + 无折扣 | 趴着，入观察清单，等市场先生抑郁 |
| 烟蒂格 | 低确定 + 大折扣 | 小仓或放弃；进则价差兑现即走 |
| 投机格 | 低确定 + 无折扣 | 永不 |

报告里本节输出：三闸各一句读数 + 一句格子判定 + **一张三闸仪表盘 ASCII 图**（放在定格结论之后）。格子结论必须与区块五的交换建议一致——若不一致，说明有一边在自欺，回头重判。

仪表盘模板（读数条用 10 格 `#`/`-` 刻度；`*XX` 用标的名缩写标进对应象限的具体位置——注意象限方位：左上黄金格、右上等待格、左下烟蒂格、右下投机格；"三股力"必须写具体，让定位可被证伪）：

```

闸门读数           弱 [----------] 强
锚   (估值地基)    [######----]  {一句读数}
竞争闸 (余生长度)   [###-------]  {一句读数}
认知闸 (域 + 量)   [####------]  {一句读数}
情绪闸 (错价程度)   [#####-----]  {一句读数}

汇总: 确定性 = 竞争闸 x 认知闸(域) => {低/中/高}
      折扣   = 情绪闸 x 认知闸(量) => {小/中/大}

                        确定性 高
                           ^
           黄金格          |          等待格

        往上顶的力: {什么会提升确定性}
```

### 五、换不换

**交换建议**：建议投资 / 建议观察 / 建议放弃

## 生成规则

1. 基于真实信息，不编造。信息不足的直接标注，不硬撑
2. 敢判断。"既可能好也可能坏"是废话，不许出现
3. 每个判断附证据——数据、引用、具体事实
4. 禁止出现：赛道很大、团队优秀、前景广阔、蓝海市场
5. 以说透为准，不计字数。两千字能说清就两千字，需要七千字就七千字
6. 中文撰写
